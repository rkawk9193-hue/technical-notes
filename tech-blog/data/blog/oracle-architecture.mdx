## 1. Oracle 메모리 구조

오라클 메모리는 크게 **SGA (System Global Area)**와 **PGA (Program Global Area)**로 나뉩니다.

### SGA (System Global Area)
모든 서버 프로세스와 백그라운드 프로세스가 공유하는 메모리 영역입니다.
*   **Shared Pool**: SQL 파싱 정보(Library Cache)와 데이터 딕셔너리 정보(Dictionary Cache) 저장.
*   **Database Buffer Cache**: 디스크에서 읽어온 데이터 블록을 캐싱.
*   **Redo Log Buffer**: 데이터 변경 사항(Redo Entry)을 저장.

### PGA (Program Global Area)
각 서버 프로세스가 독점적으로 사용하는 메모리 영역입니다.
*   **Sort Area**: 정렬 작업(Order by) 수행.
*   **Hash Area**: 해시 조인 등 수행.
*   **Session Information**: 세션 정보 저장.

---

## 2. 주요 백그라운드 프로세스

*   **DBWR (Database Writer)**: 버퍼 캐시의 변경된 블록(Dirty Buffer)을 데이터 파일에 기록.
*   **LGWR (Log Writer)**: Redo Log Buffer의 내용을 Redo Log File에 기록.
*   **CKPT (Checkpoint)**: 체크포인트 발생 시 SCN 정보를 컨트롤 파일과 데이터 파일 헤더에 갱신.
*   **SMON (System Monitor)**: 인스턴스 복구, 임시 세그먼트 정리.
*   **PMON (Process Monitor)**: 비정상 종료된 프로세스 정리, 리소스 해제.

---

## 3. 요약 (Conclusion)

Oracle의 성능 튜닝은 이 메모리 구조(SGA, PGA)의 효율적인 사용과 백그라운드 프로세스의 원활한 동작을 보장하는 것에서 시작합니다. 특히 **Library Cache** 튜닝과 **Buffer Cache** 히트율 관리는 핵심적인 요소입니다.
